# Swizzle アートディレクション 3案（WP21）

ゼロベース再設計の起点。既存テーマ（`arcade-theme.css` のネオンマゼンタ×ティール、未活用の `DESIGN_TOKENS` 516行、死蔵テーマ5種）を磨き込むのではなく、**方向性を選び直す**ための3提案。

各案は「コンセプト／パレット／タイポ／BridgeScreenモック／ゲームアセット様式／適用コスト」を含む。**コードは未変更**（`git diff` は `docs/` のみ）。

## 選定ガイド（成長戦略との対応）

**「TikTok的バイラル × 万人リーチ」を狙うなら → B. JELLY POP。**
明るくかわいいUIはスクショ映え・初見ハードルの低さで新規獲得に最も強い。ただしライト基調とPixiJSゲーム全画面の明暗ギャップ設計、実装リスク（M〜L）と最大の改修コストを許容できることが前提。

**「やり込み・リテンション・ゲーマー文化」を育てるなら → A. NEO-ARCADE。**
没入感と高級感で“常連”を作る路線。既存ネオン資産を磨いて再利用でき、現行からの移行が最短。尖りすぎるとライト層を取りこぼすリスク。フックは「上手くなりたい」動機。

**「UGC・クリエイター中心／ブランドの格」を最優先なら → C. GALLERY。**
作り手への敬意と編集的洗練で“作品が並ぶ場所”としての価値を打ち出す。実装が最軽量（S〜M）で破綻しにくく、ゲーム画面の派手さとUIの静けさが相互に引き立つ。訴求が地味・冷たいと見られるリスクを、コピー／写真／キュレーションで補う必要。

**ハイブリッド指針:** 迷う場合の安全策は **C を骨格（軽量・破綻しにくい・クリエイター訴求）に採用しつつ、アクセント運用で温度を足す**。逆に明確にグロース最優先なら B、現行資産の活用と移行コスト最小を取るなら A。

### 検証方法
3つの `proposal-*/bridge-mock.html` をブラウザで開いて並べて比較（スマホ幅390pxで成立、デスクトップでは中央に390pxのフォンフレーム表示）。
