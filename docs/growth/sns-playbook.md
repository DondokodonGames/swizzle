## 1. プラットフォーム優先順位

判断軸 = **縦型ショート動画との相性** × **ゲームプレイ録画の映え** × **着手コスト**。
Swizzle のゲームは 5–30秒・9:16 縦画面 = ショート動画とネイティブに一致する。

| 優先 | プラットフォーム | 理由 | 着手コスト |
|---|---|---|---|
| **1** | **TikTok** | 縦型ショートの本丸。アルゴリズム発見性が高くゼロフォロワーでも伸びる。5–30秒ゲームと尺が完全一致 | 無料（手動投稿）。API申請は不要（手動なら） |
| **2** | **YouTube Shorts** | 同じ縦動画素材を再利用可。検索資産化（後から流入）。TikTok と二毛作 | 無料 |
| **3** | **Instagram Reels** | 同素材を再利用。ビジュアル映えするゲーム向き | 無料（Meta Business は将来の自動化時のみ） |
| **4** | **X / Twitter** | 拡散・コミュニティ告知向け。動画より GIF/URL 共有。**API有料($100/月)なので当面は手動のみ** | 手動無料 |

> **方針**: 1本のプレイ録画を **TikTok → Shorts → Reels** に**使い回す**（同一素材の多重展開）。
> X は告知・URL誘導の補助。1プラットフォームに最適化しすぎず、まず TikTok で“当たり”を探す。

## 2. 投稿フォーマット案

### 基本フォーマット（縦型ショート, 10–15秒）
```
[0:00–0:02] フック: 一番盛り上がる瞬間 or 「3秒でわかる」テロップ
[0:02–0:12] ゲームプレイ録画（9:16・実機 or 画面収録）
[0:12–0:15] 結果画面（スコア/成功失敗）＋ CTA「君は何点？ → プロフ/概要のリンク」
```
- **素材取得**: ブラウザ/スマホの画面収録で十分。凝った編集は不要（生っぽさがショートでは有利）。
- **CTA**: プレイURL（`/play/{id}`）をプロフィールリンク/概要に。動画内では「リンクから遊べる」だけ。
- **テロップ**: 日本語＋最小英語。音は環境音 or トレンド音源（各プラットフォームのライブラリ）。

### バリエーション
- **対決/煽り型**: 「このゲーム、クリアできる人いる？」（コメント誘発）
- **ビフォーアフター型**: 失敗連発 → 成功（カタルシス）
- **ネタ紹介型**: AI生成ゲームの“変なネタ”を見せる（Swizzle 独自性）

## 3. 週次投稿カレンダー（テンプレート）

無理なく続く頻度から始める（**週3–5本 / TikTok 主軸**）。1素材を複数面に展開。

| 曜日 | TikTok | YouTube Shorts | Instagram Reels | X / Twitter |
|---|---|---|---|---|
| 月 | 新作ゲームA | （火に再利用） | — | 週の予告ポスト |
| 火 | — | ゲームA 転載 | ゲームA 転載 | — |
| 水 | 新作ゲームB（ネタ系） | — | — | ゲームB の GIF + URL |
| 木 | — | ゲームB 転載 | — | コミュニティ告知（Discord誘導） |
| 金 | 週間ベスト紹介（community §2） | 週間ベスト 転載 | 週間ベスト 転載 | 週間ベスト URL |
| 土日 | 反応の良かった素材を再投稿/Duet誘発 | — | — | フォロワー対話 |

- **計測**: 各プラットフォームのネイティブ解析（再生数・視聴維持率・プロフ遷移）を週次でメモ。
  サイト流入は WP41 の `session_start`／`signup` の増減と突き合わせる（UTM が無くても時系列で相関を見る）。
- **運用ルール**: 「完璧な1本」より「続く週3本」。当たり素材が出たらそのフォーマットを連打。

## 4. 自動化の着手条件（将来WP — 今はやらない）

手動運用で以下が揃うまで API 統合・`src/marketing/` の実装は**着手しない**。
理由: 自動化は固定費（`docs/marketing/current_status.md` 試算 **$150–220/月**, うち X API $100/月）を生む。
効果未確認のまま固定費を背負わない。

### 着手条件（すべて満たしたら自動化WPを起票）
- [ ] **手動投稿が週3本以上で 4週間継続**（運用が人手で回ることの実証）
- [ ] **少なくとも1本が“当たり”**（TikTok 再生 1万以上 等、プラットフォーム基準で）= フォーマットの型が見つかった
- [ ] **SNS経由のサイト流入が確認できる**（投稿週に WP41 `session_start`/`signup` が有意に増える）
- [ ] 自動化対象を**1プラットフォームに限定**して起票（全部一気にやらない。まず TikTok or X のどちらか）

### コスト判断メモ
| 項目 | 月額 | 着手タイミング |
|---|---|---|
| X / Twitter API (Basic) | $100 | 手動で X の効果が確認できてから**のみ** |
| Claude API（投稿文/素材生成） | $50–100 | 既存AI基盤の余剰で吸収可。新規固定費にしない |
| Cron 用サーバ（Vercel 等） | $0–20 | 自動化WP着手時 |
| **合計（フル自動化時）** | **$150–220** | 上記着手条件を満たすまで $0 を維持 |
