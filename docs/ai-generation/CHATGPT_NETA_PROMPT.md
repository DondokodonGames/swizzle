# Swizzle ネタ生成プロンプト v1.0

最終更新: 2026-04-05

---

## 目的

neta.json（ミニゲームアイデア帳）に追加するネタをChatGPTに量産してもらうためのプロンプトです。
**現在の件数: 200件（id: 1〜200）。新しいネタはid 201から開始。**

---

## ネタのフォーマット仕様

```json
{
  "id": 201,
  "title": "ゲームの短いタイトル（10文字以内）",
  "idea": "詳細な実装説明（後述のフォーマットに従う）",
  "mechanic": "tap",
  "theme": "テーマ名"
}
```

### フィールド定義

| フィールド | 型 | 説明 |
|-----------|-----|------|
| id | number | 連番（201から始める） |
| title | string | 日本語タイトル（10文字以内、ゲーム内容が分かる名前） |
| idea | string | 実装説明（下記フォーマット参照） |
| mechanic | string | "tap" \| "drag" \| "flick" \| "swipe" |
| theme | string | "ゲーム" \| "スポーツ" \| "食べ物" \| "動物" \| "乗り物" \| "パズル" \| "勉強" \| "音楽" \| "アウトドア" \| "日常" |

### idea フィールドの書き方

以下の要素を全て含む、**1〜4文の日本語**で書く：

1. **オブジェクトの外観**: 色・形・サイズ（px）・SVGの特徴
2. **初期配置**: 画面内の位置（x:0.X/y:0.Xまたは「左端」「中央下」等）
3. **ゲームメカニクス**: 操作方法・タイミング・判定
4. **勝利/失敗条件**: 何をすれば成功か、制限時間
5. **視覚フィードバック**: タップ時のエフェクト、成功時の演出

## テーマ・メカニクスの分布目安

バランスよく作るために以下の配分を目安にしてください：

### mechanic 分布（20件の場合）
- tap: 10件（50%）— 最も多い。タップ・連打・タイミング系
- drag: 4件（20%）— ドラッグ&ドロップ、なぞる系
- flick: 3件（15%）— フリック・投げる系
- swipe: 3件（15%）— 左右スワイプ、連続動作系

### theme 分布（20件の場合）
- ゲーム: 5件（人気IPやゲームのパロディ）
- スポーツ: 3件（球技、競技系）
- 食べ物: 3件（料理、食材系）
- 動物: 2件
- 乗り物: 2件
- パズル: 2件
- 勉強: 1件
- 日常: 2件（日常のちょっとした動作）

## PART A: ネタ生成システムプロンプト

*以下を Custom Instructions / システムプロンプトに設定してください*

---START---

あなたはSwizzle（スマホ向けミニゲームプラットフォーム）のゲームアイデアを大量生成する専門アシスタントです。
指示された数だけ、指定フォーマットに沿ったゲームアイデアをJSON配列で出力してください。

## 出力フォーマット

```json
[
  {
    "id": <連番>,
    "title": "<10文字以内の日本語タイトル>",
    "idea": "<実装説明>",
    "mechanic": "<tap|drag|flick|swipe>",
    "theme": "<テーマ>"
  }
]
```

## idea フィールドのルール

1〜4文で以下を全て記述する:
- SVGオブジェクトの色・形・サイズ（px単位）
- 初期配置（画面座標 x:0.X/y:0.X 形式か方向で表現）
- 操作方法（タップ/ドラッグ/フリック/スワイプ）と判定タイミング
- 勝利条件と制限時間（秒）
- 視覚フィードバック（エフェクト・演出）

## 参考例

「連打でポケモン捕まえる」→ 画面中央に赤白モンスターボール（SVG円100×100px）。10秒以内に30回タップ。タップで1.2倍拡大+赤パルス。30到達→成功エフェクト。
「バスケットボール」→ バスケットゴール（y=0.15）下でオレンジボール（70×70px）が左右往復。ゴール真下のタイミングでタップ→シュート。3回成功でクリア（10秒）。

## 制約

- 同じテーマ・メカニクスに偏らないようにする
- 既存ゲームIPの露骨なパクリは避け「〇〇風」に
- 10〜20秒で遊べるミニゲームに限定
- SVGで表現できるシンプルな見た目にする（3D不要、写実的な絵不要）
- 日本のスマホユーザーが「面白そう！」と思えるネタにする

## PART C: neta.json への追記方法

ChatGPTが出力したJSON配列を `neta.json` の `items` 配列に追記します。

### 手順

1. ChatGPTの出力（JSON配列）をコピー
2. `src/ai/v2/neta.json` を開く
3. `"items": [...]` の末尾（最後の `}` の後、`]` の前）に追加
4. `"totalCount"` の値を更新（例: 200 → 220）
5. `"description"` も更新（例: 「220件のミニゲームネタ帳」）

### バリデーション

追記後は以下を確認してください:
- [ ] id が連番になっている（重複・欠番なし）
- [ ] mechanic が "tap" | "drag" | "flick" | "swipe" のいずれか
- [ ] JSONとして正しい形式（末尾カンマなし）
- [ ] totalCount がアイテム数と一致
