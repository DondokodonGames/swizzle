# WP13: Codex自由形式バッチ — 均質化改善の実地検証

**担当**: Codex（生成）+ 人間（審査） / **依存**: WP11完了 / **想定規模**: S（Codex 10セッション + 審査30分）

---

## 背景

WP11でプロンプト/ガイドの過剰な縛りを緩めた。Codex経由の量産で実際に多様性が改善したかを、
10本のバッチ生成と人間審査で検証する。

## Codexへの指示（1ゲームごとに新しいCodexセッションで以下を貼る）

```
リポジトリの以下の2ファイルを読んでください:
- SWIZZLE_JSON_SPEC.md（フォーマット仕様。特に「品質チェックリスト」「禁止パターン集」「メカニクス・ガイド」「推奨パターン: フェーズ（状態機械）」の各節）
- docs/ai-generation/CODEX_GENERATION_GUIDE.md（ワークフロー）

そのうえで、5〜10秒で遊べるミニゲームのJSONを1本生成してください。

今回の特別な狙い: **既存のゲームと被らない、新鮮なメカニクスのゲーム**を作ること。
- src/ai/v2/neta.json から未使用のネタを1件選んでも、完全オリジナルでもよい
- メカニクスはカタログ（src/ai/v2/mechanics-catalog.json）に縛られなくてよい。
  ただし操作はタップ/スワイプ/ドラッグ/長押し（とその組み合わせ）のみ
- 多段階の構造（待つ→チャンス→判定 等）は「フェーズ・イディオム」（SPEC参照）で書くこと
- 提出前にSPECの品質チェックリストを1項目ずつ自己検証すること

出力: `project` ルートを持つ完全なJSONのみ（解説文なし）。
```

## 審査手順（人間）

1. 生成された10本のJSONを `npm run dev` → エディター → ReviewQueue（ローカルJSONアップロード）でプレイ審査
2. 各ゲームに既存タグ（動かない/タッチ反応しない/仕様違い/バランス悪い/ゲームにならない）で評価
3. **多様性の判定**: 10本中、操作タイプ・テーマ・ゲーム構造が互いに明確に異なるものが何本あるかを数える

## 成功基準

- 合格（そのまま公開可）が10本中6本以上
- 「互いに明確に異なる」ゲームが10本中7本以上（従来比で体感的に改善していること）
- フェーズ・イディオムを使ったゲームが2本以上含まれ、正しく動くこと

## 失敗時のフィードバック先

- 合格率が低い → 落ちた原因タグを WP11 のlean版プロンプトに1〜2行だけ追記（リストを再肥大化させない）
- 多様性が改善しない → CODEX_GENERATION_GUIDE の「今回の特別な狙い」節を恒久化し、過去生成タイトル一覧を渡す運用に変更
