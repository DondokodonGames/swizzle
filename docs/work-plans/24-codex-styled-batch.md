# WP24: Codex様式準拠バッチ — アートディレクションの実地検証

**担当**: Codex（生成）+ 人間（審査） / **依存**: WP23完了 / **想定規模**: S（Codex 10セッション + 審査30分）

---

## 背景

WP23でアートディレクション定義（art-direction.json）が生成・採点・ドキュメントの3者に接続された。
Codex経由の生成物が様式に従うか、見た目の一貫性が出るかを10本のバッチで検証する。

## Codexへの指示（1ゲームごとに新しいCodexセッションで以下を貼る）

```
リポジトリの以下を読んでください:
- SWIZZLE_JSON_SPEC.md（特に「アセット様式（アートディレクション）」節と「品質チェックリスト」）
- docs/ai-generation/CODEX_GENERATION_GUIDE.md

5〜10秒で遊べるミニゲームのJSONを1本生成してください。

今回の特別な狙い: **アセット様式節に厳密に従った見た目**にすること。
- 全オブジェクト・背景のSVG/画像は、様式節のパレットHEX・描法（線の太さ/塗り/形状言語）・禁止事項に従う
- 同一ゲーム内の全アセットで配色と描法を統一する
- 背景は中央を空ける（オブジェクトが乗るため）

出力: `project` ルートを持つ完全なJSONのみ。
```

加えて、AI量産パイプライン側でも `npm run ai:neta:1` を5本実行する（DALL-E経由の様式準拠を確認するため。
画像QAが automatic に様式逸脱を減点する）。

## 審査手順（人間）

1. Codex 10本を ReviewQueue（ローカルJSON）で、パイプライン5本を GameReviewQueue（審査待ちタブ+アセットプレビュー）で確認
2. **様式の一貫性チェック**: アセットプレビューのサムネイル一覧を横断で眺め、「同じプラットフォームのゲームに見えるか」を判定
3. 画像QAスコア（ai_image_score）の平均値を WP23 以前の生成分と比較

## 成功基準

- Codex 10本中7本以上が様式準拠（目視: パレット・描法が揃っている）
- パイプライン5本の ai_image_score 平均が従来分以上
- 「同じプラットフォームのゲームに見える」と判断できること（主観でよい — それがアートディレクションの目的）

## 失敗時のフィードバック先

- Codex側が従わない → SPECの様式節の記述を具体化（HEX表・SVG断片例を追加）
- DALL-E側が従わない → art-direction.json の promptFragments を強化（WP23担当者に1セクションだけ依頼）
