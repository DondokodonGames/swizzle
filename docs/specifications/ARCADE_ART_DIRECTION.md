# ARCADE ART DIRECTION — 「100円を投入したくなる」ゲーム内視覚言語

対象は**ゲーム内画面**(コードゲームのCanvas描画)。サービスUI(BridgeScreen等)のアートディレクションは
`docs/art-direction/`(GALLERY案)が正で、本書とは独立。

ゲームセンターの筐体は、電源が入っているだけで客の足を止め、100円を入れさせる。
その視覚言語を70年代から現代まで分解し、Swizzleの全ゲームに実装可能な規定として定義する。

---

## 1. アトラクトモードは3秒で遊びを売る(MUST)

ゲーセンの筐体はデモ画面で「何が起きるゲームか」を見せ続ける。ATTRACTの構成要素:

1. **ピクセルロゴ**: タイトルは `draw.text` の単発ではなく、縁取り+影+ハイライトの重ね打ち、
   または `draw.sprite` のビットマップフォントで「ロゴ」にする(bird_jumpのKOBETAN JUMP式)
2. **ゴースト実演**(v3で必須化): 手カーソル(`game.draw.hand`、WP62で追加)がAIプレイヤーとして
   実ゲームロジックを1サイクル遊んで見せる3秒ループ。成功例1回+(危険があれば)失敗例1回。
   静止一枚絵・説明文だけのATTRACTは**禁止**(PLAY_GRAMMAR_V3 §2.1)
3. **HI-SCORE 表示**: `game.best` の実値。ハードコードの偽ハイスコアは禁止(§QUALITY_STANDARD)
4. **点滅する投入誘導**: 「► 100円 投入 ◄ TAP TO START」を0.5秒周期で点滅
5. 世界観は**絵で**語る(キャラ・背景・小物)。説明テキストの掲示は禁止(PLAY_GRAMMAR_V3 §3)

## 2. 時代別スタイルパック(ゲームごとに1つ選択・MUST)

「全ゲームが同じネオン暗黒」からの脱却。テーマに合う時代を選び、ファイル冒頭にコメント宣言する:

```js
// スタイル: 80s NEON
var STYLE = { bg: [...], main: [...], accent: [...] };
```

### ① 70s MONO(インベーダー/ブロック崩し系)
- 黒背景 + 白ドット + **単色オーバーレイ帯**(ブラウン管に貼ったカラーセロハンの再現: 画面の帯ごとに緑/橙)
- 音: 矩形波の単音、無音の間が緊張を作る
- 向くテーマ: レトロSF、ミニマルな反射神経もの

### ② 80s NEON(パックマン/ギャラガ系)
- 濃紺〜黒グラデ背景 + **発光4色制限**(シアン/マゼンタ/イエロー/ホワイト)+ 対象の外周に薄い同色を重ねた疑似グロー
- 迷路/グリッド/星空の背景装飾。点滅が命
- 向くテーマ: 宇宙、迷路、ドットイート、シューティング

### ③ 90s 16bit(格ゲー/ベルトアクション/横スクロール系)
- **多色・高彩度**。背景は2〜3層(空グラデ+遠景シルエット+近景)で奥行き
- キャラは大きく、輪郭は暗色、ハイライトで立体感。表情のあるスプライト
- 向くテーマ: 冒険、動物、食べ物、キャラクターもの(**もっとも汎用**)

### ④ 2000s ARCADE POP(音ゲー/メダル/プリクラ系)
- **明るい背景**(白〜パステルグラデ)+ 原色プライマリ + 太い白縁取り文字
- 光の柱・虹・星が飛ぶ祝祭演出。「光り物」の物量で押す
- 向くテーマ: リズム、パーティ、かわいい系、2人対戦

### ⑤ NEO-RETRO(現代インディ: Downwell/ShovelKnight系)
- **限定パレット**(4〜6色)を意図的に選び、その中で完結。1色だけ強い差し色
- 大きいドット(12〜16px)で潔く。余白を恐れない
- 向くテーマ: スタイリッシュ、ホラー、ノワール、ワンボタンもの

**共通規定**: 1ゲーム = 基調3色 + アクセント2色を STYLE 宣言(スコアラー/レビューが検査可能)。
背景は `draw.gradient` + 遠景要素(黒一色はテーマ上の必然がある場合のみ)。
スキャンライン(8px)+ドット絵の「キャビネット感」は全スタイル共通で維持。

## 3. コイン投入動機の演出規定(MUST)

100円を入れさせるのは「見た目」ではなく**感情の設計**。全ゲームに以下を実装する:

| 感情 | 演出 | 実装 |
|------|------|------|
| **ジャックポット感** | スコアが爆発する瞬間を作る(コンボ倍率、ボーナスタイム、フィーバー) | `fx.popup('x8 COMBO!')` + 音程が上がっていくSE |
| **CELEBRATION** | 成功時は音と光を**飽和**させる。控えめな成功画面は禁止 | se_success + burst連発 + 点滅 + 画面全体の祝福 |
| **悔しさの言語化(ニアミス)** | 惜しい失敗ほど「あと1個!」「98%」「残り0.3秒だった」を明示 | RESULT画面に必達。失敗を次の100円に変換する装置 |
| **記録の祭り** | HI-SCORE更新は普通のクリアより派手に | NEW RECORD 点滅 + se_milestone + BridgeScreenのバッジと二段構え |
| **緊張の可聴化** | 残り時間わずか/ラストチャンスでBGMテンポ/音程が変わる | melodyのtempo切替 or 警告音ループ |

## 3.5 記号文法 — テキストなしでルールを伝える視覚言語(MUST)

指導文が禁止(PLAY_GRAMMAR_V3)のため、以下の記号を全ゲーム共通の「文法」として統一する。
プレイヤーは1本目で学んだ記号を800本すべてで再利用できる:

| 意味 | 記号 |
|------|------|
| 狙う物・押す物 | **明滅+白縁**、待機バウンス(押せる物は揺れる) |
| 危険物 | **赤系+トゲ形状**。出現の0.5〜0.8秒前に必ず予告(点滅マーカー/影/警告線) |
| ゴール | 旗・ゲート・光る出口(常時見える) |
| 残り時間 | 収縮するリング/バー(数字のみは不可) |
| 進捗 | 残りN個のアイコン列(消し込み)、伸びるゲージ、高度メーター |
| 操作ゾーン | 押せる領域は明度を上げ、縁を光らせる |
| 手本・誘導 | ゴーストハンド(`game.draw.hand`)+タップリング |

テーマ(台帳の theme_jp)はこの記号の**上に着せる皮**であり、記号の意味を壊してはならない
(例: お菓子テーマでも危険物はトゲ+赤系を保つ)。

## 4. キャラクターの最低要件

- 主役(操作対象 or 目標物)は**顔がある/名前が付けられる**モチーフにする(目2ドットで十分)
- 2〜4フレームで「生きている」動き(待機で揺れる、移動でバタバタ、成功でジャンプ)
- 失敗時にキャラがリアクションする(倒れる/驚く/焦げる) — 記号ではなく演者にする

## 5. 監査

- `npm run games:smoke` のスクリーンショット・コンタクトシートで人間が確認:
  「スタイルパックが判別できるか」「ATTRACTだけで遊びが分かるか」「キャラに顔があるか」
  「指導文が写っていないか」「危険物が記号文法(§3.5)通りか」
- `CodeQualityScorer` の audioVisual / goalEndings 軸が sprite / gradient / best / popup を静的検査
  (WP63 でテキストレス検査と attract_motion 実行時検査が加わる)
