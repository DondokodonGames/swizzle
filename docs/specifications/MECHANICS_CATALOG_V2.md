# MECHANICS CATALOG v2 — ショートゲームの動詞40

メイドインワリオ(5秒マイクロゲームの動詞設計)・マリオパーティ(1台対戦ミニゲーム)・
アーケード(コイン1枚の緊張と祭り)・プレイアブル広告/ハイパーカジュアル(初見3秒で理解できる1動作)
の主要な遊びを、Swizzle サンドボックス API v2 で実装可能な形に体系化したカタログ。

用途:
- **新規ネタ(801〜2000)のメカニクス割当** — 未使用/低使用のIDを優先して多様性を作る(現状: タップ798/スワイプ201/ホールド5/ドラッグ0 という偏りの是正)
- アップロード時の `// @mechanic:` ヘッダー値(user_games.mechanic → フィードの連続回避インターリーブ)
- 書き換え/新規セッションへの実装指針(必要API・フィードバック設計)

全メカニクスは API v2 で実装可能なことを確認済み。レシピは [SANDBOX_API_V2.md](./SANDBOX_API_V2.md)。

## A. タイミング系(押す瞬間が勝負)

| ID | 動詞 | 遊び | 必要API | フィードバック設計 |
|----|------|------|---------|-------------------|
| timing_one_shot | 一発で押す | 動く針/ゲージが最良点に来た瞬間タップ | onTap | 精度で good のtext を変える(PERFECT!/GOOD/MISS) |
| timing_window | 窓で押す | 開閉する当たり判定窓の間にタップ | onTap | 窓の可視化+成否で good/bad |
| mash | 連打 | N秒間に規定回数叩く(綱引き/脱出) | onTap | 1打ごと se_tap+ゲージ増、規定到達で milestone |
| alternate_tap | 交互連打 | 左右ボタンを交互に(走る/登る) | onTap(x判定) | 正しい交互=good、同側連打=bad |
| cooldown_tap | 我慢して押す | 連打禁止。クールダウン明けだけ有効 | onTap | 早押しは bad(ペナルティ明示) |
| hold_charge | 溜めて離す | 長押しでパワー充填→離して発射 | onPress/onRelease/input.pressing | チャージゲージ+音程上昇、離した強さを表示 |
| hold_duration | 押し続ける | 指定時間ぴったり押して離す | onPress/onRelease | 経過表示、誤差で PERFECT/GOOD/MISS |
| freeze | 動くな | 「見られている」間は触らない(だるまさん) | onPress(禁止検出) | 禁止中に触れたら即 bad+シェイク |
| rhythm | リズムで押す | BGMの拍に合わせてタップ | onTap + audio.melody | 拍ズレms で判定段階、コンボ表示 |

## 実装優先度ガイド(バリエーション是正)

現800本は A系(タップ)に極端に偏っている。**新規/書き換えの割当は以下を優先:**

1. **D系ジェスチャー**(trace/slice/rub/rotate) — v2のonMoveで初めて可能。体感が新しい
2. **B系発射**(slingshot/flick/drop) — 「引いて放す」はプレイアブル広告の王道
3. **C系追従**(dodge/camera_climb/balance) — 押しっぱなし入力で初めて可能。bird_jump系
4. **G系対戦** — 1台2人はSNS動画映え(マリオパーティ的シェア動機)
5. A/F系は既存資産が厚いので、演出強化(§H複合)を中心に

ネタ割当時は `npm run ai:status` の多様性レポートで mechanic 分布を確認し、
占有率が15%を超えた動詞への新規割当を止める。
